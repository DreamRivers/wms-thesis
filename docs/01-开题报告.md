## 一、课题来源与背景

随着电子商务的快速发展,仓储管理成为电商企业核心竞争力的重要组成部分。传统人工仓库管理方式存在效率低、错误率高、库存不透明等问题,难以满足现代电商对"快速响应、零差错、可追溯"的严苛要求。

近年来,以 Spring Boot、Vue.js 为代表的前后端分离开发技术日趋成熟,Java 生态在企业级应用开发中具有显著优势。本课题拟设计并实现一套适用于中小型电商企业的仓储物资管理系统(以下简称 WMS),涵盖商品管理、库位管理、入库管理、出库管理、库存管理、盘点管理、库存预警等核心功能。

## 二、国内外研究现状

### 2.1 国外现状
国外 WMS 系统发展较为成熟,代表产品包括 Manhattan Associates、HighJump、Körber 等。这些系统功能完善但部署成本高,适合大型企业。

### 2.2 国内现状
国内 WMS 厂商如富勒信息、科箭软件、旺店通等已占据较大市场,产品多基于 SaaS 模式。但对中小企业而言,定制化与价格仍是痛点,基于开源框架自研的轻量级 WMS 仍有较大应用空间。

### 2.3 现有方案的不足
- 商业系统价格昂贵,中小企业难以承担;
- 部分开源系统功能简陋,缺少完整的审批流和库存预警机制;
- 单体架构难以适应业务扩展。

## 三、研究内容与拟解决的关键问题

### 3.1 主要研究内容
1. 基于 RBAC 模型实现 4 类角色(管理员/仓库管理员/部门负责人/普通员工)的权限管理;
2. 设计商品 SKU + 库位 + 批次 三维度的库存模型,保证库存数据一致性;
3. 实现入库单、出库单的完整状态机流转;
4. 出库申请采用"部门负责人 → 仓管"两级审批流;
5. 定时任务实现低库存与临期预警;
6. 前后端分离架构,后端 RESTful API + 前端 SPA。

### 3.2 拟解决的关键问题
- **库存并发安全问题**:采用 `SELECT ... FOR UPDATE` 行级锁 + 事务保证原子性;
- **单据状态合法性**:通过状态机 + Service 层校验;
- **操作审计**:基于 Spring AOP + 自定义 `@Log` 注解,自动记录操作日志;
- **可扩展性**:模块化设计,业务代码与框架解耦。

## 四、研究方法与技术路线

### 4.1 技术栈
- 后端:Spring Boot 2.7.18 + MyBatis-Plus 3.5.5 + Sa-Token 1.37 + MySQL 8.0 + Redis 7
- 前端:Vue 3.4 + Vite 5 + Element Plus 2.5 + ECharts 5 + Pinia + Axios
- 工具:IDEA / VS Code / Maven 3.8+ / Node 18+ / Git
