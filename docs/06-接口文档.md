# 接口文档

> 自动生成地址:启动后端后访问 [http://localhost:8080/doc.html](http://localhost:8080/doc.html)
> 框架:Knife4j 4.5.0 (基于 Swagger 2.0)

## 一、统一约定

### 1.1 鉴权
所有 `/api/**` 请求(除登录外)需在 Header 携带:
```
Authorization: <Sa-Token 登录返回的 token>
```

### 1.2 统一返回格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... },
  "timestamp": 1717200000000
}
```

### 1.3 错误码
| 码 | 含义 |
|---|---|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录 |
| 403 | 无权限 |
| 500 | 业务异常 |
| 1001 | 库存不足 |
| 1002 | 状态非法 |
| 1003 | 数据不存在 |
| 1004 | 重复提交 |

## 二、核心接口

### 2.1 登录

**POST** `/auth/login`

请求:
```json
{
  "username": "admin",
  "password": "123456",
  "captchaKey": "uuid",
  "captcha": "1234"
}
```

响应:
```json
{
  "code": 200,
  "data": {
    "token": "xxxxxxxx",
    "tokenName": "Authorization",
    "userId": 1,
    "username": "admin",
    "realName": "系统管理员",
    "roles": ["ADMIN"]
  }
}
```

### 2.2 入库单分页

**GET** `/inbound/order/page?pageNum=1&pageSize=10&orderNo=RK&status=PENDING`

响应:
```json
{
  "code": 200,
  "data": {
    "total": 100,
    "list": [
      { "id": 1, "orderNo": "RK202606010001", "status": "PENDING", "totalQty": 50, "totalAmount": 5000.00, ... }
    ]
  }
}
```

### 2.3 入库单保存(草稿)

**POST** `/inbound/order/save`

请求:
```json
{
  "supplierId": 1,
  "warehouseId": 1,
  "orderType": 1,
  "remark": "测试",
  "items": [
    { "goodsId": 1, "locationId": 1, "batchNo": "B001", "planQty": 50, "unitPrice": 100.00 }
  ]
}
```

### 2.4 出库申请

**POST** `/outbound/order/apply`

### 2.5 出库审批

**POST** `/outbound/order/approval/handle`

请求:
```json
{ "orderId": 1, "step": 1, "pass": true, "remark": "同意" }
```

### 2.6 库存查询

**GET** `/stock/list/page?pageNum=1&pageSize=20&goodsId=1`

### 2.7 仪表盘

**GET** `/report/dashboard`

响应:
```json
{
  "code": 200,
  "data": {
    "totalStock": 1570,
    "todayIn": 3,
    "todayOut": 5,
    "warningCount": 2,
    "stockSkuCount": 10
  }
}
```

## 三、典型业务流

### 3.1 入库完整流程

```bash
# 1. 登录
curl -X POST http://localhost:8080/auth/login -H "Content-Type: application/json" -d '{"username":"admin","password":"123456"}'

# 2. 保存草稿
curl -X POST http://localhost:8080/inbound/order/save -H "Authorization: <token>" -H "Content-Type: application/json" -d '{"supplierId":1,"warehouseId":1,"items":[...]}'

# 3. 提交
curl -X POST http://localhost:8080/inbound/order/submit/1 -H "Authorization: <token>"

# 4. 审核
curl -X POST http://localhost:8080/inbound/order/audit -H "Authorization: <token>" -H "Content-Type: application/json" -d '{"id":1,"pass":true,"remark":""}'

# 5. 执行
curl -X POST http://localhost:8080/inbound/order/execute/1 -H "Authorization: <token>" -H "Content-Type: application/json" -d '[{"id":1,"actualQty":50}]'

# 6. 完成(写库存)
curl -X POST http://localhost:8080/inbound/order/complete/1 -H "Authorization: <token>"
```

### 3.2 出库两级审批

```bash
# 部门负责人审批
curl -X POST http://localhost:8080/outbound/order/approval/handle -d '{"orderId":1,"step":1,"pass":true,"remark":""}' -H "Authorization: <dept_token>"

# 仓管审批
curl -X POST http://localhost:8080/outbound/order/approval/handle -d '{"orderId":1,"step":2,"pass":true,"remark":""}' -H "Authorization: <wh_token>"

# 发货
curl -X POST http://localhost:8080/outbound/order/ship/1 -H "Authorization: <wh_token>"
```
