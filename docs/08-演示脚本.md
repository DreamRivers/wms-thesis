# 5 分钟演示脚本

> 用于毕业答辩现场演示。**总时长严格控制在 5 分钟内**。
> 演示前请确保:① 后端已启动 ② 前端已启动 ③ MySQL 与 Redis 已就绪 ④ 已登录相关账号。

## 演示前准备

```
✅ MySQL 已导入 wms_db 库与种子数据
✅ Redis 已启动(端口 6379)
✅ 后端服务:cd wms-backend && mvn spring-boot:run
✅ 前端服务:cd wms-frontend && npm run dev
✅ 打开浏览器:http://localhost:5173
```

## 演示流程(5 分钟)

### 第 1 阶段:登录与首页(45 秒)

**0:00 - 0:15** 打开浏览器,展示登录页
- "这是基于 Spring Boot + Vue 3 的电商仓储物资管理系统"
- 输入 admin / 123456 + 验证码,登录

**0:15 - 0:45** 展示首页
- 4 张统计卡:库存总量、今日入库、今日出库、预警数
- 2 张折线图:近 7 天入库/出库趋势
- "接下来演示完整的入库出库流程"

### 第 2 阶段:仓管创建入库单(60 秒)

**0:45 - 1:00** 切换 wh001 账号登录
- "仓库管理员 wh001 登录,负责商品和入库管理"

**1:00 - 1:30** 进入"入库管理 → 入库单列表"
- 点击"新建入库单"
- 选择供应商(华为技术)、仓库(主仓)
- 添加明细:选择"华为手机"、库位 A-01-01、批次 B001、计划数量 100、单价 5000
- 点击"保存草稿"

**1:30 - 1:45** 提交审核
- 列表中找到刚创建的单据,点击"提交"
- 状态变为"待审核"

### 第 3 阶段:管理员审核(45 秒)

**1:45 - 2:00** 切回 admin 账号
- 进入"入库管理 → 入库单审核"
- 看到待审核单据,点击"通过",输入"同意"
- 状态变为"已审核"

**2:00 - 2:15** 切换 wh001 账号
- 回到入库单列表,点击"执行"
- 录入实收数(默认等于计划数),状态变"执行中"

**2:15 - 2:30** 点击"完成"
- 弹窗提示"完成后将写入库存",点击确定
- 状态变为"已完成"
- 强调"此时库存已自动增加,流水已记录"

### 第 4 阶段:库存查询与出库流程(90 秒)

**2:30 - 2:45** 进入"库存 → 实时库存"
- 找到"华为手机",数量已变为 150(原 50 + 刚入库 100)
- "库存实时更新,数据准确"

**2:45 - 3:00** 切换 emp001 账号(普通员工)
- 进入"出库管理 → 出库申请"
- 选择出库类型"领用",仓库"主仓"
- 出库原因"项目部需要 5 台测试机"
- 添加明细:华为手机、库位 A-01-01、批次 B001、数量 5
- 点击"提交申请"

**3:00 - 3:30** 切换 dept001 账号(部门负责人)
- 进入"出库管理 → 出库审批"
- 默认 Tab"待部门审批"看到刚提交的申请
- 点击"通过",输入"同意,用于测试"
- 状态变为"审批中"

**3:30 - 3:45** 切换 wh001 账号(仓管)
- 进入"出库管理 → 出库审批"
- 切换 Tab"待仓管审批"
- 点击"通过",输入"库存充足"
- 状态变为"已审核"

**3:45 - 4:00** 回到"出库单列表",点击"发货出库"
- 状态变为"已发货",库存已扣减
- 点击"完成" → 状态"已完成"

### 第 5 阶段:库存预警与盘点(60 秒)

**4:00 - 4:20** 进入"库存 → 实时库存"
- 华为手机数量已变为 145(150-5)
- "库存准确,实现了出库自动扣减"

**4:20 - 4:40** 库存预警
- 进入"库存 → 库存预警"
- "系统每小时自动扫描,这里展示低库存与临期商品"
- 演示标为已读、全部已读

**4:40 - 4:55** 盘点
- 进入"库存 → 盘点管理"
- 点击"新建盘点单",选择"主仓"+"全盘"
- 演示创建一个盘点单
- 点击"详情"录入实盘数(故意与系统数差 1 件)
- 演示"完成并调整",库存自动修正

### 第 6 阶段:报表与结束(5 秒)

**4:55 - 5:00** 回到首页,展示更新后的数据
- "至此完成完整的入库→库存→出库→盘点→预警闭环,谢谢各位老师!"

---

## 演示注意事项

1. **语速**:每阶段控制在规定时间内,如超时简化细节
2. **账号切换**:提前打开多个浏览器标签页或使用隐身模式,避免反复登出
3. **网络**:确保后端已正常启动,首次接口调用可能稍慢
4. **数据**:每次演示前可重置数据库,保证初始状态一致
5. **意外处理**:如遇接口异常,可点击"重试"或提前准备好的截图
